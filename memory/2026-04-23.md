**Technical wins:**

- ✅ `.focus()` instead of `.click()` on body editor (avoids hitting signature area at bottom)

- Use `page.keyboard.type()` to preserve signatures (not `.fill()`)
- No em dashes in email content (Matthew's preference)
- Lead with wearable panic button badges (not generic "school safety")
- Say "set up a meeting" not "quick call"

**Helper functions created:**
- `dismissPopups(page)` - Escape + common close/cancel buttons
- `closeFilterDropdown(page, screenshotDir)` - Escape + pill re-click
- `extractContactInfo(page)` - Returns `{email, firstName, districtName}` from preview panel
- `closeComposeWindow(page, screenshotDir, contactLabel)` - Find Email header → click last button (X)

**Next steps:**

## Compose Window Close Challenge #blocker

**Problem:** Email compose window X button proving difficult to target reliably with selectors.

**What we tried:**
1. Generic close button selectors (`button[aria-label="Close"]`) - failed, not specific enough
2. Brute-force all close buttons - failed, clicked wrong elements
3. Filter compose by Subject + Send button, then find close inside - failed
4. Email header text → parent → last button - failed (Playwright retried 58× "waiting for element to be visible")

**Visual location confirmed by Matthew (screenshot):**
- Navy blue header bar at top of compose window
- Text "Email" on left side of header
- X close button is rightmost icon in that navy header (far right edge)
- Clear visual target but selector/locator strategies not reaching it

**Next approaches to try:**
- Coordinate-based click using Subject field bounding box as anchor
- Look for the teal "Cathy Gent" contact preview panel as a landmark, compose is adjacent
- OCR or image-based click if all selector strategies fail
- Investigate if X is in shadow DOM or iframe

**Impact:** Blocking progression to second contact - can't batch process until compose close works reliably.
